Change ringer

This will enable the functionality to change the ringing destination. If there is a headset plugged into the computer, it is easy to miss a call due to the device ringing on the headset. Changing the ringer will allow a call to ring on the computer’s physical speaker even with a headset plugged in.

  1. Click the change ringer button.
  2. Choose the appropriate option for selecting the ringer output:
    1. If "Default" is selected, the system will use the default audio device.
    2. If "Communications Default" is selected, this will be used for audio related to communication apps (e.g., calls, meetings).
    3. If "Speakers" is selected, audio will be played through connected speakers or headphones using the Realtek audio driver.
    4. If “Headphones” are selected, the device will ring through the connected headphones.

Step 1: Click the change ringer button.

Step 2: Choose between default, communications default, speakers or Headphones.

Example

Filters

  1. From: A date with time representing the start of your query time range. No information older than this date is used.
  2. To: A date with time representing the end of your query time range. No information newer than this date is used.
  3. User: The user whose activity the report should display. Leaving this blank will display data for all users.
  4. Channel Types: Specific Channels the information should be limited to. Selecting Channel ‘A’ and ‘B’ for example will result in data related to those Channels. Specifying no Channels means all channels will be included.

Headlines

  1. Total Interactions: The total amount of incoming interactions offered to the agent or outbound interactions made by the agent. Note that inbound interactions that failed to offer, for instance, the agent not answering on time or it being a voice interaction without the agent having a connected voice device, are not counted.
  2. Answered: The total amount of Answered Interactions. Inbound and Outbound.
  3. Answered Inbound: The total amount of Answered Inbound Interactions.
  4. Answered Outbound: The total amount of Answered Outbound Interactions.
  5. Rejected: The total amount of Inbound Interactions Rejected by the user. Where the user has physically rejected the interaction.
  6. Missed: The total amount of Inbound Interactions Offered to the user that they did not answer within the departments offering timeout and the user was put on "Not Responding". Thereby being a Missed Interaction.
  7. Note: This excludes Outbound interactions which are not answered.
  8. Avg Answer: The average time it takes for the user to answer an interaction.
  9. Avg Handle Time: Average handle time. The average time the user spends handling an Interaction. This includes time communicating and wrapping up.
  10. Presentations: Amount of dispositions applied that are presentations.
  11. DMC: Amount of dispositions applied that are decision-making contacts.
  12. DNC: Amount of dispositions applied that are Do Not Calls.
  13. Sales: Amount of dispositions applied that are sales.
  14. Talk Time: Total duration the agent has spent communicating. Note this sum does not include time spent in the wrap-up.
  15. Wrap Time: Total duration agent has spent in wrap up. This is the time between the end of the interaction and the moment the agent dispositions the interaction if a disposition is required.
  16. Idle Time: The total duration an agent has spent in the idle status. This means the agent is not on queue or paused.
  17. Pause Time: The total duration an agent has spent in the pause status. This can be expanded to display a breakdown of the various time spent in various pause statuses. Select the toggle called "Breakout pause reasons".
  18. Pending Time: Shows the sum of the (Waiting for Customer) measurements used in the response times report.
  19. Warm Transfer: Total count of warm transfers.
  20. Blind Transfer: Total count of blind transfers.
  21. Not Responding: Total time the user spent in "Not Responding". Interactions were offered to the user, that didn't answer before the Offering Timeout was reached. The user state is then changed to "Not Responding".
  22. Dialer %: The average duration an agent has spent in the following states: on queue, communicating or wrap up based on the total amount of time they have spent in all states while logged in.
  23. Occupied %: The average duration an agent has spent on interactions (communicating and wrap-up) based on the total amount of time they have spent in all states while logged in.
  24. CSAT Score: The average CSAT a user has received across the interactions where CSAT was rated. The CSAT rating is from 1-5 Stars, where 1 is the lowest rating and 5 is the highest rating.
  25. CSAT Count: How many Interactions received a CSAT rating
  26. Days of Attendance: Number of days an agent was present and on queue and answered an interaction. Calculated daily each time an agent logs in, goes on queue and answers an Interaction.
  27. User Status: What percentage and time the user spent if certain statuses
  28. Call Outcome: A Disposition has additional options that can be selected further defining what the Disposition represents. These selection options are available in Settings > Dispositions > Select a Disposition. This displays a breakdown of Dispositions which have these options selected.
  29. Conversion (%): The percentage of decision-making contacts converted into sales.
  30. Presentation Conversion (%): The percentage of your total calls dispositioned as presentations.
  31. All Dispositions: Graph visualizing applied dispositions per channel type.

User Interactions Outputs

  1. Label: Disposition name.
  2. Calls: Amount of voice interactions.
  3. Emails: Amount of email interactions.
  4. Chats: Amount of web chat interactions.
  5. SMS: Amount of SMS interactions.
  6. Socials: Amount of social interactions (Facebook, Instagram, …).

  10. DMC: Whether the disposition is a DMC.

  14. Average Duration: The average duration of the interactions.
  15. Unique Contacts: Amount on unique contacts.

Log out

Click the log out button. Logging out is crucial for maintaining data security, protecting sensitive information, and ensuring system integrity. By logging out, users prevent unauthorised access to confidential client data and reduce the risk of data breaches. It also helps the system track user activity accurately, which is essential for auditing and compliance.
