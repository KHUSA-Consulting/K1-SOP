  13. Fields that need to be populate as provided by Coligo:
    1. Department Details and Basic Routing Settings
      • Department Name: Identifies the department, e.g., "SOP".
      • Offering Timeout: Sets how long to offer an incoming call or interaction to a user. Example: 10 seconds.
      • Priority: Determines interaction priority if multiple departments have waiting interactions; lower numbers have higher priority.
      • Manually Close Department: Allows for manual closure, ignoring schedule settings.
    2. Grouping and Management
      • Extension: Specifies an internal extension for direct department reach.
      • Department Group: Group for reporting; helps analyze departmental performance.
      • Managers: Assigns department managers for oversight.
      • Alert Notification Email: Specifies an email for alert notifications.
    3. Transfer and Caller ID Settings
      • Disallow Warm/Blind Transfer: Prevents warm (consulted) or blind (direct) transfers to this department.
      • Force Customer's CLI on Transfer: Ensures the customer’s Caller Line Identification (CLI) is preserved on transfer if supported by the carrier.
    4. Call Handling and Routing
      • On Hangup Self Service Script: Runs a self-service script when an agent hangs up.
      • Force Callback Request Routing: Directs callback requests to this department's default route, removing customization options.
      • Dialer Categories and Lead Disposition: Sets up categories for dialer leads and tracks lead disposition on non-dialer calls.
    5. Automatic Assignments and Compliance
      • Auto Assign on Accept: Auto-assigns contacts to users upon answer if unassigned.
      • TCPA Compliant: Prompts agents to confirm contact method for dialing, ensuring compliance.
    6. Alerts and Notifications
      • Inbound Alert Configuration: Executes workflows if a queue surpasses specified wait times.
      • Agent Not-Responding Alerts: Sends alerts if agents repeatedly miss calls.
      • Unanswered Interactions Alert: Alerts managers if a threshold of unanswered calls is reached.
      • Long Voice Call Alert: Notifies managers for calls exceeding a set duration.
    7. Disposition and Call Recording
      • Dispositioning: Requires agents to label calls with wrap-up codes post-interaction.
      • Record Telephone Calls: Enables automatic call recording.
      • Cloud Storage Recording: Uploads recordings to a cloud endpoint, optionally filtering by disposition.
    8. Call and Contact Management
      • Auto Answer/Dial: Auto-answers or dials calls, including automated voice calls.
      • Inbound and Outbound Scripts: Provides agents with scripts for customer interactions.
    9. Outbound Communication and Consent
      • Blocked Outgoing Channels: Prevents outbound interactions on specific channels, useful for training or testing.
      • Block Outbound Communication Without Consent: Blocks communication if the contact hasn’t consented to that channel.
    10. Attachment and Message Handling
      • Convert Images to PDFs: Converts image attachments to PDFs before sending.
      • SMS and WhatsApp Template Schedule: Restricts the hours for sending SMS or WhatsApp templates.
    11. Overflow and Call Routing
      • Overflow Configurations: Routes calls to other destinations if the department is closed or if no agents are available.
      • Overflow Timeout: Time to wait before moving a call to overflow; set to 0 to disable.
    12. Conversation Snooze and Inactivity Handling
      • Snooze Settings: Allows pausing a conversation while awaiting a customer response, routing back after timeout.
      • Inactivity Conversation Snooze: Automatically snoozes conversations after inactivity, routing back after timeout.
    13. Voicemail and Caller ID Customization
      • Department Voicemail: Enables voicemail if no one answers the call.
      • Prefix Inbound Caller ID: Adds a prefix to the caller ID, useful for identifying the department.
    14. Secure Upload and Custom Fields
      • Secure Upload: Sets up secure upload options for sharing documents safely.
      • Custom Fields: Auto-assigns custom fields relevant to the department for data consistency.

  11. Fields that need to be populated to create a new routing rule (As provided by Coligo):
    1. SOP (Standard Operating Procedure) Name
      • Purpose: Assigns a meaningful name to the rule that helps identify its purpose.
      • Example: “High-Priority Support Escalation”
    2. Rule Type
      • Option: "Inbound"
      • Purpose: Indicates that this rule applies to incoming interactions, rather than outgoing ones.
    3. Apply Rule on Incoming Interactions
      • Option: "Yes" (or checked)
      • Purpose: Confirms that this rule will only act on interactions coming in, not on replies or outbound messages.
    4. Channel Types
      • Purpose: Specifies which communication channels (like email, chat, SMS) the rule should monitor.
      • Example: Email, Callback.
    5. Channels to Apply Rule To
      • Purpose: Narrow down the rule to specific channels or even subchannels (such as a specific phone line for callbacks).
      • Example: Callback for support.
    6. Priority
      • Option: "0" is the highest priority, as the rule with the lowest value takes precedence.
      • Purpose: If multiple rules match, the system uses the priority number to determine which rule to apply first.
      • Example: Set to “0” for critical escalation rules.
    7. Matching Conditions
      • Purpose: Defines specific criteria for the rule to trigger based on parts of an incoming interaction.
      • Regular Expression: Allows advanced pattern matching in various fields. If enabled, it applies regex criteria to specific interaction fields like Subject, From, To, or Body.
      • Subject: Matches keywords or phrases in the subject line.
      • From: Matches the sender's email address.
      • To: Matches the recipient's address.
      • Body: Matches specific words or patterns within the message content.
      • Example: A rule to filter messages with “urgent” in the subject line or from specific domains in the "From" address.
    8. Referral Source
      • Purpose: Filters messages by their origin, such as specific websites or campaigns.
    9. Language
      • Purpose: Filters messages based on the language, useful for multilingual support systems.
    10. Forward Email
      • Purpose: Specifies an email address to forward all matching interactions.
      • Field: Enter the email address where matching emails should be sent.
    11. Routing
      • Purpose: Defines where and how the interaction should be directed if it meets the rule conditions.
      • Field: Redirect Endpoint to specify the endpoint for routing the message, such as a team or individual.
    12. No Conversation
      • Purpose: If checked, prevents creating a conversation thread for the interaction. It’s useful if the rule is set up to ignore or discard certain messages.
    13. Force New Conversation
      • Purpose: Forces a new conversation thread regardless of existing threads with the sender.
      • Use Case: Useful when each interaction needs to be handled as a new query, such as high-priority requests.
      • Field Population Requirements:
      • To configure a rule, typically, the following fields should be populated based on the intent:
      • SOP Name: Essential for identifying the rule.
      • Priority: To establish rule order in case of overlap.
      • Matching Conditions: Select relevant fields like Subject, From, To, Body, etc., based on the criteria you wish to use for the rule.
      • Forward Email or Redirect Endpoint: Necessary if the rule intends to forward or redirect the message.
      • No Conversation or Force New Conversation: Optional based on whether you need to skip or enforce a new thread for each message.

  13. Fields that need to be populated (as provided by Coligo):
    1. Basic Information
      • First Name / Last Name
      • Job Title: Enter the user's job title (e.g., "Support Agent").
      • Contact Number: Primary telephone number for the user.
    2. WhatsApp Notification Telephone
      • Purpose: Allows sending notifications over WhatsApp if enabled for the organization.
    3. Email
      • Purpose: The primary email is also used as the user’s login username.
    4. Enabled
      • Purpose: Controls access to the user account. Uncheck to disable access.
    5. Country
      • Purpose: Specifies the user's location for applying local dialing rules.
      • Example: South Africa.
    6. Extension
      • Purpose: Assigns an internal extension number for the user.
    7. Departmental and Access Configuration
    8. Departments / Restricted Departments / Recording Access Departments
      • Purpose: Defines which departments the user belongs to, restricts specific departments for inbound interactions, and specifies recording access if restrictions apply.
    9. Roles
      • Purpose: Assigns roles, like "Standard," to define permissions and access levels.
    10. Groups / Supervisor for Groups
      • Purpose: Lists groups the user is a member of and groups they supervise.
    11. Remote Access
      • Purpose: Allows access from any IP if IP restrictions are active.
    12. User Type
      • Purpose: Assigns the user a "Standard" type license.
    13. Call Handling and Device Setup
    14. Device
      • Purpose: Specifies the device for incoming calls (e.g., WebRTC device)
    15. Legacy Email
      • Purpose: Used for Single Sign-On (SSO) and legacy systems to match user data, such as in Salesforce.
    16. Divert Destination
      • Purpose: Redirects calls to a specific number or extension instead of voicemail.
    17. Disable Auto Logout
      • Purpose: Prevents automatic logouts for the user.
    18. Force Inbound Transfer Department
      • Purpose: Routes direct transfers to a specified department. Defaults to the last known department if left blank.
    19. Default Currency
      • Purpose: Used for interactions that involve payments or currencies.
    20. Voicemail and Interaction Handling
    21. Voicemail Enabled
      • Purpose: Enables voicemail access for the user.
    22. Response Timeout / Not Responding Auto Recovery
      • Purpose: Sets a timeout for unanswered interactions and automatically reassigns the user to the queue.
    23. Idle Timeout / Idle Auto Recovery
      • Purpose: Sets a timeout for idle status and automatically places the user back in the queue.
    24. Enforce Set Caller ID / Enabled Forced Caller ID
      • Purpose: Forces a specific Caller ID for all outbound calls if required.
    25. Incoming Conversation Handling
      • Purpose: Configures automated messages to present to customers for incoming conversations.
    26. Channel Priority / Channel Restriction
      • Purpose: Defines the priority and restriction of channels for incoming interactions.
    27. Automatic Answer Timeout
      • Purpose: Sets the time after which incoming calls are automatically answered.
    28. Custom Fields
      • Purpose: Allows additional information like Mobile and LinkedIn URL for personal or professional data.
    29. Field Population - To fully configure this user:
      • Basic Information: Fill in First/Last Name, Contact Number, Email, and Country.
      • Access Settings: Select Enabled, Extension, Departments, Roles, and Groups.
      • Device & Call Handling: Assign Device, Divert Destination, Response Timeout, Voicemail, and any necessary Caller ID settings.
      • Channel and Priority: Configure Incoming Conversation Handling, Channel Priority, and restrictions.

These configurations allow tailored routing, automated notifications, and managed communication flows for the user within the system.
